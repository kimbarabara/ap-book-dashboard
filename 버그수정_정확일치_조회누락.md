# 버그 수정 요청 — 정확 일치 조회에서도 교재 행 누락 (값 정규화 문제)

## 1. 문제 현상

반별 교재 조회에서 **공통 / 6학년 / 08월**을 선택해 조회했는데,
아래 행이 결과에 나오지 않음:

| 관 | 반(학년) | 교재명 | 월 |
|---|---|---|---|
| 공통 | 6학년 | A Summer to Die | 08월 |

관·반·월이 모두 등록값과 동일한 조건인데도 누락됨 → 매칭 규칙 이전에 **값 비교 자체가 실패**하고 있음.

## 2. 조사할 원인 후보 (우선순위 순)

코드를 확인하여 아래 중 실제 원인을 찾아 수정할 것. 여러 개가 동시에 해당될 수 있음.

### 후보 1: 월 표기 정규화 부재
- 데이터의 월 값(`08월`)과 조회 드롭다운의 월 값(`8월` 등)의 표기가 다르면 문자열 비교가 실패함.
- **수정**: 월 비교 시 양쪽 모두 숫자로 정규화 후 비교. `"8월"`, `"08월"`, `8`, `"8"` 모두 8로 취급. `공통`은 기존대로 와일드카드.

### 후보 2: 공백/보이지 않는 문자
- 시트 셀 값에 앞뒤 공백, 탭, 전각 공백 등이 섞여 있으면 비교 실패.
- **수정**: 관/반/월/교재명 등 모든 비교 대상 값에 대해 비교 전 `trim()` (가능하면 연속 공백 정리 포함) 적용. 조회 값과 데이터 값 양쪽 모두.

### 후보 3: 관 컬럼의 와일드카드 표기 — `공통` (확정된 사실)
- **실제 DB시트의 관 컬럼은 와일드카드를 `공통`으로 표기하고 있음** (스크린샷으로 확인됨). 즉 관과 월 모두 와일드카드 표기가 `공통`으로 동일함.
- 기존 스펙 문서에 "관 와일드카드 = `공용`"이라고 적힌 부분이 있다면 그건 표기 착오이며, **실데이터 기준은 `공통`**.
- 코드가 관 컬럼에서 `공용`만 특별 취급하고 있다면 이것이 이번 누락의 유력한 원인.
- **수정 방침**: 관 컬럼에서 `공통`을 와일드카드(모든 관에 포함)로 취급. 방어적으로 `공용`이라는 값이 들어와도 동일하게 와일드카드로 취급 (둘 다 인정).
  - 이렇게 하면 어떤 관을 선택하든(1관, 2관, …) `공통` 관 행이 항상 포함됨.

### 후보 4: 드롭다운 옵션 값과 표시 텍스트 불일치
- 드롭다운의 표시 텍스트는 `공통`/`6학년`/`08월`인데 실제 전달되는 value가 다른 값(빈 문자열, 코드값 등)인 경우.
- **수정**: 드롭다운 value와 데이터 값이 같은 정규화 규칙을 거치도록 통일.

## 3. 실제 데이터 특성 (스크린샷 확인 사항, 구현 시 참고)

- 관 와일드카드 표기: `공통`
- 월 값 형식: `03월`, `04월`, `08월` 등 **두 자리(0 포함) + "월"** 형식 사용. 단 한 자리 입력(`8월`)이 섞일 가능성도 있으므로 숫자 정규화 필수.
- 월 와일드카드 표기: `공통` (관과 동일한 단어이지만 컬럼이 다르므로 컬럼별로 판정하면 충돌 없음)
- **교재명이 빈 행 존재**: `공통 / 6학년 / (교재명 없음) / 03월`처럼 교재명이 비어 있는 행이 있음. 이런 행은 조회 결과에서 제외하되(빈 카드 방지), 에러로 중단되지 않아야 함.

## 4. 수정 원칙

- 정규화(트림, 월 숫자화)는 **비교 시점에만** 적용하고, 시트 원본 데이터는 변경하지 말 것.
- 정규화 로직은 반별 교재 조회 / 과목별 교재 조회 / 교재 낱개 검색이 **공통 함수를 사용**하도록 한 곳에 구현 (탭마다 따로 구현 금지).
- 기존 규칙(공용·공통 와일드카드, N학년 매칭, 미등록 표시)은 유지.
