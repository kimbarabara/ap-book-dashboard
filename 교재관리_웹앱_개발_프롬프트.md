# 학원 교재 관리 대시보드 웹앱 — 개발 프롬프트

> 아래 내용을 그대로 복사하여 AI 개발 툴(Claude, GPT, v0 등)에 입력하세요.

---

## [역할]

당신은 Google Apps Script와 웹 프론트엔드에 능숙한 풀스택 개발자입니다. 아래 요구사항에 따라 **구글 시트를 데이터베이스로 사용하는 학원 교재 관리·청구 대시보드 웹앱**을 개발해 주세요.

## [기술 스택 및 배포 방식]

- **백엔드/호스팅:** Google Apps Script 웹앱 (`doGet` + `google.script.run`) — 별도 서버·비용 없음
- **데이터베이스:** 기존 구글 스프레드시트를 그대로 DB로 사용 (시트 수정 시 웹앱에 자동 반영)
- **프론트엔드:** HTML + CSS + JavaScript 단일 파일 구조, PC 화면(1280px 이상) 기준 레이아웃
- **인증:** 구글 계정 로그인 (Apps Script 배포 설정: "액세스 권한 - 도메인/특정 사용자", `Session.getActiveUser().getEmail()`로 접속자 식별)
- **PDF 생성:** 브라우저 인쇄 기반(`window.print()` + 인쇄용 CSS) 또는 jsPDF — 한글 폰트 깨짐 없이 표가 깔끔하게 출력되어야 함

## [데이터베이스 구조 — 구글 시트 4개 탭]

### 1. 교재DB (마스터 테이블)
| 필드 | 설명 |
|---|---|
| 과목 | 교재의 과목 분류 |
| 교재명 | 고유 키(Primary Key) |
| 출판사 | |
| ISBN | |
| 청구가격 | 숫자 |
| 학년 | |
| 대상 월 | |

### 2. 반별교재셋팅 (커리큘럼 매핑 테이블)
| 필드 | 설명 |
|---|---|
| 관 | 캠퍼스/분원 이름. **'공용'이면 모든 관에 해당** |
| 학년/반이름 | |
| 교재명 | 교재DB의 교재명과 연결되는 외래 키 |
| 대상 월 | '공통' 또는 '7월', '8월' 등 특정 월 |
| 메모 | |

### 3. 사용자권한 (접근 제어 테이블) — 신규 생성
| 필드 | 설명 |
|---|---|
| 이메일 | 구글 계정 이메일 |
| 역할 | '관리자' 또는 '강사' |

- 이 시트에 등록된 이메일만 웹앱 접속 가능. 미등록 이메일 접속 시 "접근 권한이 없습니다. 관리자에게 문의하세요." 안내 화면 표시.
- 역할에 따라 관리자/강사 화면을 다르게 렌더링.
- **보안 필수:** 화면에서 숨기는 방식이 아니라, 서버 측(Apps Script) 함수에서 역할을 검증한 뒤 데이터를 내려줄 것. 강사 계정으로는 교재DB·반별교재셋팅 시트 원본 데이터에 접근할 수 없어야 함(시트 파일 자체는 강사에게 공유하지 않음).

### 4. 데이터 수정 방식
- 교재DB와 반별교재셋팅의 추가/수정/삭제는 **관리자가 구글 시트에서 직접 수행** (웹앱 내 편집 화면 없음).
- 웹앱은 페이지 로드 또는 조회 시마다 시트에서 최신 데이터를 읽어와 항상 최신 상태를 반영.

## [화면 구성]

### 공통 상단
- 앱 제목 + 로그인된 사용자 이메일 + 역할 배지(관리자/강사) 표시

### 섹션 A — 교재 낱개 검색 (관리자·강사 공통)
- 입력창 1개: **교재명 또는 ISBN**으로 검색 (부분 일치 지원)
- 검색 결과를 카드 형태로 출력: 과목, 교재명, 출판사, ISBN, 청구가격, 학년, 대상 월

### 섹션 B — 반별 교재 조회 (관리자·강사 공통)
- 드롭다운 3개: **[관 선택] [반(학년) 선택] [사용 월 선택]** — 옵션 목록은 반별교재셋팅 시트에서 동적으로 생성
- 조회 결과 테이블 컬럼:
  - **관리자:** [체크박스] [NO] [과목] [교재명] [출판사] [ISBN] [청구가격] [사용 월]
  - **강사:** [NO] [과목] [교재명] [출판사] [ISBN] [청구가격] [사용 월] — 체크박스·총액 없음
- **관리자 전용:** 테이블 상단 우측에 큰 글씨로 "★ 청구 총액: 0원" 표시, 체크박스 선택에 따라 실시간 변동
- 테이블 하단에 **[PDF로 저장]** 버튼 (관리자·강사 공통)

## [핵심 비즈니스 로직]

### 로직 1 — 반별 교재 필터링 및 조인 (⚠ '공용' 관 처리 주의)
사용자가 [관 = X], [반 = Y], [사용 월 = Z]를 선택하면:
1. 반별교재셋팅에서 다음 조건을 **모두** 만족하는 행을 필터링:
   - `관 == X` **또는** `관 == '공용'` ← 관에 상관없이 학년별로 동일하게 쓰는 원서는 '공용'으로 등록되어 있으므로, 어떤 관을 선택해도 해당 학년이면 함께 표시되어야 함
   - `학년/반이름 == Y`
   - `대상 월 == '공통'` **또는** `대상 월 == Z`
2. 필터링된 행의 `교재명`을 키로 교재DB와 조인하여 과목, 출판사, ISBN, 청구가격을 결합해 테이블에 표시.
3. 교재DB에 없는 교재명이 있으면 해당 행에 "교재DB 미등록" 표시(에러로 중단하지 말 것).

### 로직 2 — 실시간 청구 총액 계산 (관리자 전용)
- 각 행 왼쪽 체크박스 기본값: 해제(unchecked)
- 체크된 행의 청구가격만 합산하여 "★ 청구 총액"에 즉시 반영. 전부 해제 시 0원.
- "전체 선택/해제" 체크박스를 헤더에 제공.

### 로직 3 — PDF 저장
- **관리자:** 체크박스로 **선택된 항목만** PDF에 포함.
- **강사:** 현재 조회된 목록 **전체**를 PDF에 포함.
- PDF 양식: 별도 로고·날짜 없이 **표만 깔끔하게** 출력. 조회 조건(관/반/월)은 표 위에 한 줄로 표기.
- 한글이 깨지지 않아야 하며, A4 세로 기준으로 열 너비가 적절히 맞춰져야 함.

### 로직 4 — 접근 제어
- 앱 진입 시 `Session.getActiveUser().getEmail()`로 이메일 확인 → 사용자권한 시트와 대조.
- 미등록: 차단 화면. 강사: 섹션 A + 섹션 B(조회·PDF만). 관리자: 전체 기능.
- 모든 서버 함수(`google.script.run`으로 호출되는 함수)는 실행 시마다 역할을 재검증할 것.

## [샘플 데이터 요청]

테스트용 목데이터를 코드에 포함하되, 실제로는 시트에서 읽어오는 구조로 작성:
- 관: '1관', '2관', '공용'
- 반: '4학년', '5학년'
- 4학년 공통 교재 2권(그중 1권은 관='공용'인 원서), 4학년 8월 교재 1권
- 사용자권한: 관리자 1명, 강사 1명
