# 버그 수정 요청 — 반별 교재 조회에서 "N학년" 등록 교재 누락

## 1. 문제 현상

반별 교재 조회에서 반(학년)을 `6M/6P`처럼 구체적인 반 이름으로 선택해 조회하면,
학년 단위(`6학년`)로 등록된 교재가 결과에 **누락됨**.

현재 매칭이 반 이름의 정확한 문자열 일치로만 동작하는 것으로 보임.
`6학년`으로 등록된 교재는 6이 들어간 모든 반(6M, 6P, 6S, 6M/6P 등)의 조회 결과에 포함되어야 함.

## 2. 재현 예시

교재 데이터(발췌):

| 관 | 반(학년) | 교재명 | 월 |
|---|---|---|---|
| 공통 | 6학년 | A Summer to Die | 08월 |
| 1관 | 6M/6P | 능률보카 어원편 (개정 전) | 공통 |
| 1관 | 6M/6P | Longman Academic Reading 4 | 공통 |

**1관 / 6M/6P / 8월** 조회 시 기대 결과: 위 3건 모두 포함.
- 1행: 관 `공통`(모든 관에 포함) + 반 `6학년`(학년 6 → 6M/6P에 포함) + 월 08월 일치
- 2·3행: 관 일치 + 반 정확 일치 + 월 `공통`(모든 월에 포함)

**현재 결과**: 1행(A Summer to Die)이 누락됨. ← 이것이 버그.

## 3. 수정할 매칭 규칙 (반/학년 축)

조회 시 선택한 반 이름을 `선택반`, 교재 행에 등록된 반 값을 `등록반`이라 할 때, 다음 중 하나면 포함:

1. `등록반` == `선택반` (정확 일치, 기존 동작)
2. `등록반`이 `N학년` 형식이고, `선택반`에서 추출한 학년 숫자에 N이 포함되는 경우
   - **학년 숫자 추출 규칙**: `선택반`을 `/`로 분리한 각 토큰에 대해, 토큰이 **숫자로 시작하는 경우에만** 그 선행 숫자를 학년으로 추출.
     - `6M/6P` → 토큰 6M, 6P → 학년 {6}
     - `5S` → 학년 {5}
   - **알파벳으로 시작하는 반은 학년 없음으로 처리** (숫자가 있어도 레벨 표기이지 학년이 아님):
     - `G2`, `G2.5`, `G3` (Grammar Lv2 등) → 학년 추출 안 함 → 어떤 `N학년` 등록 교재와도 매칭되지 않음
   - 예: 등록반 `6학년` → 선택반 `6M`, `6P`, `6S`, `6M/6P` 모두 매칭
   - 예: 등록반 `6학년` → 선택반 `5S` 매칭 안 됨
   - 예: 등록반 `2학년` → 선택반 `G2`, `G2.5` 매칭 안 됨 (G의 숫자는 레벨)

관 축(`공용` = 모든 관 포함), 월 축(`공통` = 모든 월 포함)의 기존 규칙은 그대로 유지하며,
세 축의 판정은 독립적으로 AND 결합 (관 매칭 AND 반 매칭 AND 월 매칭).

## 4. 주의사항

- 기존 `공용` 관 / `공통` 월 포함 로직을 깨뜨리지 말 것.
- "교재DB 미등록" 표시 로직(에러로 중단되지 않음)도 그대로 유지.
- 관-학년-과목 조회(과목별 교재 조회) 탭에도 동일한 반/학년 매칭 로직을 쓰고 있다면 같은 수정 적용.
- 반 이름이 `/`로 연결된 복합 반(예: 6M/6P)은 토큰별로 학년을 추출해 합집합으로 취급.
