# 학원 교재 관리 대시보드

Google Apps Script + Google 스프레드시트를 데이터베이스로 사용하는 교재 관리/청구 대시보드 웹앱입니다.
개발 요구사항 원문은 [`교재관리_웹앱_개발_프롬프트.md`](./교재관리_웹앱_개발_프롬프트.md)를 참고하세요.

## 파일 구성

| 파일 | 설명 |
|---|---|
| `Code.gs` | Apps Script 백엔드 (시트 읽기, 권한 검증, 조회/검색 로직) |
| `index.html` | 프론트엔드 (HTML+CSS+JS 단일 파일) |
| `appsscript.json` | Apps Script 프로젝트 매니페스트 (웹앱 실행/접근 설정) |

## 1. 스프레드시트 준비

기존에 사용하던 구글 스프레드시트를 그대로 사용합니다. 아래 탭이 필요합니다.

1. **교재DB** — 컬럼: `과목, 교재명, 출판사, ISBN, 청구가격, 학년, 대상 월`
2. **반별교재셋팅** — 컬럼: `관, 학년/반, 교재명, 대상 월, 메모`
3. **사용자권한** — 컬럼: `이메일, 구분, 이름, 등록일` (구분 값은 `관리자` / `직원` / `뷰어`)
4. **감사로그** — 컬럼: `시각, 수행자이메일, 동작유형, 대상, 변경내용요약` (교재DB 등록·권한 변경 이력이 자동으로 쌓이는 탭. 직접 입력할 필요 없음)
5. **업데이트** — 컬럼: `날짜, 분류, 내용, 표시여부` (관리자가 이 탭에 행을 직접 추가하는 것만으로 대시보드에 공지가 뜹니다)
   - `분류` 값은 `기능추가` / `기능수정` / `데이터반영` / `공지` 중 하나 (탭별로 색이 다르게 표시됨).
   - `표시여부`가 `Y`인 행만 노출됩니다. 작성 중이거나 내리고 싶은 공지는 `N`으로 바꾸면 숨겨집니다.

> 탭 이름이나 컬럼 이름이 다르면 `Code.gs` 최상단의 `SHEET_NAMES` / `BOOK_DB_FIELDS` / `CLASS_SETTING_FIELDS` / `USER_PERMISSION_FIELDS` / `AUDIT_LOG_FIELDS` / `UPDATE_LOG_FIELDS` 상수만 실제 이름에 맞게 수정하면 됩니다. 컬럼은 순서가 아니라 이름으로 찾으므로 시트 안에서 컬럼 순서를 바꿔도 문제없습니다.

### 기존에 쓰던 스프레드시트를 업그레이드하는 경우 (1회만)

이미 **사용자권한** 탭에 `이메일, 구분` 두 컬럼만 있는 상태로 운영 중이었다면, Apps Script 편집기에서 함수 선택 드롭다운에서 `runFeatureUpgradeSetup` 을 선택하고 ▶ 실행 버튼을 한 번 눌러주세요.
- **사용자권한** 탭에 `이름` / `등록일` 컬럼이 없으면 자동으로 추가하고, 기존 행들의 등록일을 오늘 날짜로 채웁니다.
- **감사로그** 탭이 없으면 자동으로 생성합니다.
- **업데이트** 탭이 없으면 자동으로 생성하고, 시작 안내용 공지를 한 줄 채워 넣습니다.
- 이미 반영되어 있다면 다시 실행해도 안전합니다(중복 실행 무해).

## 2. Apps Script 프로젝트에 코드 붙여넣기

1. 스프레드시트 상단 메뉴에서 **확장 프로그램 → Apps Script** 클릭 (컨테이너 바인딩 스크립트가 생성됩니다).
2. 기본 생성된 `Code.gs` 내용을 모두 지우고, 이 저장소의 `Code.gs` 내용을 그대로 붙여넣습니다.
3. 왼쪽 `+` → **HTML** 파일 추가 → 파일명을 정확히 `index` 로 지정 → 이 저장소의 `index.html` 내용을 그대로 붙여넣습니다.
4. (선택) 프로젝트 설정 톱니바퀴 → `appsscript.json` 표시를 켜고, 이 저장소의 `appsscript.json` 내용으로 교체합니다.

## 3. 샘플 데이터로 시트 세팅 (선택)

시트에 아직 데이터가 없다면, Apps Script 편집기 상단 함수 선택 드롭다운에서 `setupSampleData` 를 선택하고 ▶ 실행 버튼을 누르세요.
- 이미 존재하는 탭은 건드리지 않고, **없는 탭만** 자동으로 생성 + 샘플 데이터를 채웁니다.
- 처음 실행 시 권한 승인 화면이 뜨면 본인 계정으로 승인하세요.
- 생성되는 샘플: 관(`1관`,`2관`,`공용`), 반(`4학년`,`5학년`), 4학년 공통 교재 2권(그중 1권은 `공용`), 4학년 8월 교재 1권, 관리자 1명 + 직원 1명(`admin@example.com` / `teacher@example.com` — 실제 이메일로 반드시 교체하세요).

## 4. 사용자권한 시트에 실제 이메일 등록

**사용자권한** 탭에 실제로 접속을 허용할 구글 계정 이메일과 역할(`관리자`/`직원`/`뷰어`)을 등록하세요. 이 시트에 없는 이메일은 접속 시 "접근 권한이 없습니다" 화면이 표시됩니다. (배포 후에는 관리자가 웹앱 안의 **권한 관리** 화면에서 직접 추가/변경/회수할 수도 있습니다 — 8번 참고.)

## 5. 웹앱 배포

1. Apps Script 편집기 우측 상단 **배포 → 새 배포**.
2. 유형 선택에서 **웹 앱** 선택.
3. 설정:
   - **다음 사용자 권한으로 실행:** `앱에 접근하는 사용자` (필수 — 이래야 `Session.getActiveUser().getEmail()`로 접속자별 이메일을 식별합니다)
   - **액세스 권한:**
     - 회사/학원이 **Google Workspace 도메인**을 쓴다면 `도메인 내 사용자` 선택 권장.
     - **개인 Gmail 계정**만 쓴다면 `Google 계정이 있는 모든 사용자` 선택 (로그인은 필수이며, 실제 접근 허용 여부는 **사용자권한** 시트가 서버 단에서 다시 검증하므로 안전합니다).
4. **배포** 클릭 후 나오는 웹 앱 URL을 접속자들에게 공유합니다.
5. `Code.gs`나 `index.html`을 수정한 뒤에는 **배포 → 배포 관리 → 편집(연필 아이콘) → 새 버전** 으로 다시 배포해야 변경사항이 실제 URL에 반영됩니다.

## 6. 권한별로 보이는 화면

| 기능 | 관리자 | 직원 | 뷰어 |
|---|---|---|---|
| 세 탭 조회 (전체 관/반/월 범위) | ✅ | ✅ | ✅ |
| 교재 단가 표시 (낱개 카드 + 조회 결과) | ✅ | ✅ | ❌ |
| 체크박스 + ★ 청구 총액 | ✅ | ✅ | ❌ |
| PDF 저장 (브라우저 인쇄) | ✅ | ✅ | ✅ (가격 없는 화면 그대로) |
| 미등록 교재 등록 요청 | — | ✅ | ✅ |
| 미등록 교재 요청함 (처리) | ✅ | ❌ | ❌ |
| 권한 관리 화면 | ✅ | ❌ | ❌ |
| 감사 로그 조회 | ✅ | ❌ | ❌ |
| 교재DB 시트 연결 버튼 / 접속자 현황 | ✅ | ❌ | ❌ |
| "이 교재를 쓰는 반" 역조회 | ✅ | ✅ | ✅ |
| 업데이트 공지 조회 | ✅ | ✅ | ✅ |

관리자에게는 화면 우측 상단에 **"⚙ 관리자 화면"** 버튼이 따로 노출되며, 요청함/권한관리/감사로그/접속현황은 이 버튼을 눌러야 들어가는 별도 화면으로 분리되어 있습니다(조회 탭 3개는 그대로 메인 화면에 남아 있음). 직원·뷰어에게는 이 버튼 자체가 서버 단에서부터 내려가지 않습니다.

가격 관련 필드는 **서버 응답 단계에서부터** 뷰어에게 아예 내려주지 않습니다(프론트에서 숨기는 방식이 아님). 모든 관리 기능(요청 승인/반려, 권한 변경, 감사 로그 조회 등)도 서버 함수 진입 시점에 역할을 매번 재검증합니다.

## 7. 사용 방법

- **교재 낱개 검색:** 교재명 또는 ISBN 일부만 입력해도 카드 형태로 결과가 표시됩니다. 카드의 **"이 교재 쓰는 반 보기"** 버튼을 누르면 이 교재를 사용하는 관/반/사용 월 목록이 뜹니다(전 권한 공통, 가격 정보 없음).
- **반별 교재 조회:** 관 / 반(학년) / 사용 월을 모두 선택 후 조회.
  - `공용` 관으로 등록된 교재는 어떤 관을 선택해도 항상 포함됩니다.
  - `공통` 월로 등록된 교재는 어떤 월을 선택해도 항상 포함됩니다.
  - 교재DB에 등록되지 않은 교재명은 "교재DB 미등록"으로 표시됩니다(에러로 중단되지 않음). 이때 **직원/뷰어**에게는 같은 행에 **[등록 요청]** 버튼이 뜹니다 — 눌러서 교재명(수정 가능)/출판사/ISBN/메모를 적어 제출하면 관리자에게 전달됩니다. 같은 교재명으로 이미 대기 중인 요청이 있으면 버튼 대신 "이미 요청됨" 배지가 표시됩니다. 요청이 반려되면 다음 접속 시 화면 상단에 반려 사유가 안내됩니다.
  - **관리자·직원**만 체크박스와 실시간 "★ 청구 총액", 청구가격 컬럼을 볼 수 있습니다.
- **과목별 교재 조회:** (구 "관-학년-과목 조회") 관 / 반(학년) / 과목 선택 후 조회. 월 조건 없이 전체 월 대상입니다.
- **신규 교재 등록 요청 (직원·뷰어):** 교재 낱개 검색 카드 옆 **"+ 신규 교재 등록 요청"** 버튼 — 아직 반별교재셋팅에도 없는 교재를 미리 요청할 때 씁니다. 교재명을 입력하고 "확인"을 누르면 교재DB와 대기 중 요청을 자동으로 검색해 비슷한 게 있으면 먼저 보여주고, "아니요, 새 교재입니다"를 눌러야 나머지 입력(출판사/ISBN/메모/사용 예정 반)으로 넘어갈 수 있습니다. 처리 흐름(요청함/중복 방지/반려 안내)은 위의 [등록 요청]과 동일하게 합쳐집니다.
- **PDF로 저장:** 브라우저 인쇄 기능을 이용합니다. 관리자·직원은 **체크된 항목만**, 뷰어는 **조회된 전체 목록(가격 없이)**이 인쇄/저장됩니다. 인쇄 대화상자에서 "PDF로 저장"을 선택하면 PDF 파일로 저장됩니다.
- **업데이트 소식 (전 권한):** 헤더의 **"📢 업데이트 소식"** 버튼 — **업데이트** 시트의 공지를 최신순으로 보여줍니다. 최근 7일 이내 새 공지가 있으면 버튼에 배지가 뜹니다.
- **미등록 교재 요청함 (관리자):** 직원/뷰어가 제출한 등록 요청(미등록 발견형 + 신규 요청형 모두)이 여기 모입니다. 대기 중 요청 수가 화면 제목 옆 배지 및 헤더의 "⚙ 관리자 화면" 버튼 배지에 표시됩니다. **승인**을 누르면 요청 내용이 미리 채워진 교재DB 등록 폼이 열리고, 제출 시 실제 교재DB 시트에 새 행이 추가되며 요청은 자동으로 "처리됨"이 됩니다. **반려**를 누르면 사유를 입력해 요청자에게 안내할 수 있습니다.
- **권한 관리 (관리자):** 시트를 직접 열지 않고도 화면에서 이메일+권한(+이름)을 입력해 계정을 추가하거나, 드롭다운으로 기존 계정의 권한을 바로 바꾸거나, "회수" 버튼으로 계정을 삭제할 수 있습니다. 등록일이 함께 표시되므로 알바생 교체 시 회수 누락을 줄이는 데 도움이 됩니다.
- **감사 로그 (관리자):** 교재DB 등록(요청 승인)과 권한 부여/변경/회수가 발생할 때마다 자동으로 기록됩니다. 기간(시작일/종료일)과 수행자 이메일로 필터링해 조회할 수 있습니다.

## 8. 데이터 수정

교재DB / 반별교재셋팅 데이터의 추가·수정·삭제는 기본적으로 **구글 시트에서 직접** 수정합니다(교재DB는 요청함 승인을 통해 웹앱에서 추가하는 것도 가능 — 7번 참고). 웹앱은 조회할 때마다 시트에서 최신 데이터를 다시 읽으므로 별도 동기화 없이 즉시 반영됩니다.

다만 **드롭다운 목록(관/반/월/과목)은 페이지를 열 때 한 번만 불러오므로**, 시트에 새 관/반/월/과목 값을 추가한 뒤에는 접속자가 웹앱 페이지를 새로고침(F5)해야 새 옵션이 보입니다.

## 9. 새 사용자 추가하기 — 관리자와 직원/뷰어는 필요한 공유 권한이 다릅니다

한 사람을 새로 추가하려면 아래를 **모두** 등록해야 접속이 됩니다. 하나라도 빠지면 접속이 안 됩니다.

1. **사용자권한** 탭에 그 사람의 구글 계정 이메일과 역할(`관리자`/`직원`/`뷰어`)을 한 행 추가 (또는 관리자가 웹앱의 **권한 관리** 화면에서 추가).
2. 스프레드시트 **공유** 설정에서 일반 액세스는 `제한됨`을 유지한 채, 그 이메일을 개별로 추가:
   - **직원 / 뷰어**: 권한을 **뷰어**로 지정하면 됩니다. (조회 기능은 읽기만 하고, 미등록 교재 등록 요청은 시트가 아니라 스크립트 자체 저장소에 저장되므로 뷰어 권한으로 충분합니다.)
   - **관리자**: 권한을 **편집자**로 지정해야 합니다. 관리자만 수행하는 동작(미등록 교재 요청 승인 → 교재DB에 새 행 추가, 권한 관리 화면에서 사용자권한 시트 수정, 감사 로그 기록)은 실제로 스프레드시트에 쓰기 때문입니다. 관리자를 뷰어 권한으로만 두면 이 기능들을 쓸 때 "이 작업을 실행할 권한이 없습니다" 같은 오류가 납니다.
   - (웹앱이 `이 앱에 액세스하는 사용자` 권한으로 실행되므로, 접속자 본인이 시트에 최소한 위 권한을 갖고 있어야 스크립트가 동작합니다. `링크가 있는 모든 사용자`로 설정하면 안 됩니다 — 원본 데이터가 공개로 노출됩니다.)

그 사람에게는 **스프레드시트 URL이 아니라 웹앱 URL만** 전달하고, 반드시 사용자권한에 등록한 그 이메일 계정으로 로그인한 상태에서 접속하도록 안내하세요.

원본 데이터를 더 안전하게 보호하려면 교재DB / 반별교재셋팅 / 사용자권한 / 감사로그 탭을 우클릭 → **시트 숨기기** 처리하는 것도 추천합니다. 뷰어 권한만 있는 사람은 숨긴 시트를 다시 표시할 수 없습니다.
